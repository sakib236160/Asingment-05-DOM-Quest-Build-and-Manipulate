Q1: How does JavaScript handle dynamic content insertion in the DOM?

Answer:JavaScript enables dynamic content insertion in the DOM through methods such as createElement, appendChild, and insertBefore. When you need to add new elements, such as a list item or a button, you can create an element using createElement, set its attributes or content, and then insert it into the DOM tree using methods like appendChild. This allows you to dynamically modify the structure of a webpage based on user interaction or data changes, without needing to reload the entire page.

Q2: What is the difference between querySelector and querySelectorAll in the DOM?

Both querySelector and querySelectorAll are used to select elements in the DOM using CSS selectors, but they differ in their return values. querySelector returns the first matching element it finds, making it ideal for cases where you only need one element, like selecting a single button or input field. In contrast, querySelectorAll returns a NodeList of all matching elements, even if multiple elements match the selector. This makes querySelectorAll useful when working with lists of elements, such as selecting all the items in a menu or every paragraph on a page.

Q3: How does JavaScript handle event propagation and what is the difference between capturing and bubbling?

Event propagation in JavaScript refers to the way events are handled in the DOM hierarchy, specifically in the phases of capturing, target, and bubbling. In the capturing phase, events are first captured by the outermost elements and then propagate inward. In the bubbling phase, events start at the target element and bubble outward to the root. By default, most events use bubbling, but event listeners can be set to capture events instead by using the capture option in addEventListener. Understanding these phases allows developers to control the flow of events, such as preventing default actions or handling events in a more granular way.
